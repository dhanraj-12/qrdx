## List of Subprocessors

qrdx uses the following third-party service providers to help deliver our service. We carefully select our subprocessors to ensure they meet our standards for security and data protection.

### Vercel

- **Purpose:** Server hosting and edge infrastructure
- **Location:** US
- **Reference:** [DPA](https://vercel.com/legal/dpa)

### Neon

- **Purpose:** Relational database (PostgreSQL)
- **Location:** US
- **Reference:** [Security](https://neon.tech/docs/security/security-overview)

### Upstash

- **Purpose:** Redis caching & queuing infrastructure
- **Location:** US, EU, Singapore
- **Reference:** [Trust Center](https://trust.upstash.com)

### Better Auth

- **Purpose:** Authentication and session management
- **Location:** Self-hosted
- **Reference:** [Documentation](https://www.better-auth.com/)

### PostHog

- **Purpose:** Product analytics and feature flags
- **Location:** US, EU
- **Reference:** [DPA](https://posthog.com/docs/privacy/dpa)

### Polar

- **Purpose:** Payment processing and subscriptions
- **Location:** EU
- **Reference:** [Privacy Policy](https://polar.sh/legal/privacy)

### Cloudflare

- **Purpose:** CDN and image optimization
- **Location:** Global
- **Reference:** [DPA](https://www.cloudflare.com/cloudflare-customer-dpa)

### Anthropic

- **Purpose:** AI-powered features and LLM
- **Location:** US
- **Reference:** [Trust Center](https://trust.anthropic.com)

### GitHub

- **Purpose:** Source code management and OAuth authentication
- **Location:** US
- **Reference:** [DPA](https://docs.github.com/en/site-policy/privacy-policies/github-data-protection-agreement)

### Google

- **Purpose:** OAuth authentication and Google Drive integration
- **Location:** US
- **Reference:** [DPA](https://cloud.google.com/terms/data-processing-addendum)
